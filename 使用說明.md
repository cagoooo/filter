## 1. 系統簡介

**成績篩選系統**是一套專為學校評量業務設計的前端工具，用於從 Excel/CSV 成績資料中，依照年級、科目、排名比例或固定名額，自動篩選出符合條件的學生名單。

### 核心功能

| 功能 | 說明 |
|------|------|
| 多科目匯入 | 支援國文、英文、數學三科成績分別上傳 |
| 智慧欄位偵測 | 自動識別學號、姓名、班級、分數等欄位 |
| 輔助名單 | 可上傳「現職學生」（優先）與「特殊學生」（排除）名單 |
| 批次篩選 | 支援一次設定多年級、多科目的篩選條件 |
| 兩種篩選模式 | 比例篩選（前 X%）或固定名額（前 N 名） |
| 結果查看 | 列表模式與分組模式，支援搜尋與排序 |
| 匯出功能 | 支援 Excel (.xlsx) 與 CSV 格式匯出 |
| 自動儲存 | 所有資料自動儲存於瀏覽器，重新整理不遺失 |

### 系統架構特點

- **純前端應用**：所有運算皆在瀏覽器本地執行，不需後端伺服器
- **零資料外洩風險**：上傳的成績資料不會傳送至任何外部伺服器
- **離線可用**：只需瀏覽器即可使用（建置後）

## 2. 快速開始（開發環境）

### 環境需求

- **Node.js** v18 以上（建議 v20+）
- **pnpm** v8 以上（必須，不可使用 npm/yarn）

### 安裝步驟

```bash
# 1. 進入專案目錄
cd H:\Filter

# 2. 安裝所有相依套件
pnpm install

# 3. 啟動開發伺服器（成績篩選系統）
pnpm run dev
```

啟動後，瀏覽器開啟 **http://localhost:5173** 即可使用。

### 建置正式版本

```bash
# 建置所有套件（型別檢查 + 打包）
pnpm run build

# 建置輸出位置
# artifacts/grade-filter/dist/public/
```

### 僅啟動 grade-filter 模組

```bash
cd artifacts/grade-filter
pnpm run dev
```

## 4. 步驟一：匯入成績資料

### 4.1 上傳成績檔案

頁面提供五個上傳區域：

| 上傳區域 | 用途 | 必填 |
|---------|------|------|
| 國文成績 | 各年級國文科成績 Excel/CSV | 建議填 |
| 英文成績 | 各年級英文科成績 Excel/CSV | 建議填 |
| 數學成績 | 各年級數學科成績 Excel/CSV | 建議填 |
| 現職學生名單 | 優先學生名單（出現時標記為 ⭐ 優先）| 選填 |
| 特殊學生名單 | 排除學生名單（即使符合條件也不列入）| 選填 |

**上傳方式**：
- 拖曳檔案至上傳區域
- 點擊上傳區域選擇檔案

**支援格式**：`.xlsx`, `.xls`, `.csv`

### 4.2 上傳預覽與欄位確認

上傳後，系統自動偵測欄位並顯示**預覽畫面**，包含：

- **偵測到的欄位對應**（例：A欄 → 學號、B欄 → 姓名）
- **前 3 筆資料預覽**
- 若偵測不確定，會顯示**手動調整選項**

#### 手動調整欄位

若自動偵測結果不正確，可點擊每個欄位旁的下拉選單手動指定：

```
A欄: [學號 ▼]    B欄: [姓名 ▼]    C欄: [班級 ▼]    D欄: [分數 ▼]
```

可選項目：學號、姓名、年級、班級、座號、身分證字號、國文分數、英文分數、數學分數、忽略

確認後點擊「**套用設定**」重新解析資料。

### 4.3 資料驗證警告

系統會自動偵測以下問題並顯示警告：

| 警告類型 | 說明 |
|---------|------|
| **重複學生** | 相同身分證字號出現多次 |
| **異常分數** | 分數超出合理範圍（0–150）|
| **科目缺失** | 學生出現在某科但另一科找不到對應資料 |
| **跨科目 ID 不符** | 同名學生在不同科目檔案中身分證字號不一致 |

> 警告僅供參考，不影響後續操作，可選擇忽略。

### 4.4 完成匯入

確認資料無誤後，點擊「**前往篩選設定**」進入步驟二。

## 5. 步驟二：設定篩選條件

### 5.1 新增篩選規則

點擊「**新增條件**」按鈕新增一條篩選規則，每條規則包含：

| 欄位 | 說明 |
|------|------|
| **年級** | 選擇要篩選的年級（一年級 / 二年級 / 三年級）|
| **科目** | 國文 / 英文 / 數學 |
| **模式** | 比例（前 X%）或 名額（前 N 名）|
| **數值** | 百分比（1–100）或固定名額（正整數）|

**範例規則**：

```
二年級 + 國文 + 比例 + 20%  →  篩選二年級國文前 20% 的學生
三年級 + 數學 + 名額 + 15   →  篩選三年級數學前 15 名的學生
```

### 5.2 批次新增（推薦）

點擊「**批次新增**」展開批次面板，可一次為多個年級設定相同條件：

```
勾選年級：[✓ 一年級] [✓ 二年級] [✓ 三年級]
科目：國文
模式：比例
數值：20%

→ 一次新增 3 條規則
```

### 5.3 預覽篩選結果數量

每條規則右側會顯示**符合人數預覽**，讓您在執行前先確認數量是否合理。

### 5.4 執行篩選

確認所有規則後，點擊「**執行篩選**」按鈕，系統依以下邏輯處理：

```
1. 排除「特殊學生名單」中的學生
2. 依分數從高到低排序
3. 依設定的比例/名額取出符合學生
4. 將「現職學生名單」中的學生標記為 ⭐ 優先
5. 其餘符合條件者標記為一般
6. 不符合但在現職名單中者也保留（優先標記）
```

## 6. 步驟三：查看與匯出結果

### 6.1 統計圖表

頁面頂端顯示統計長條圖：
- **各年級人數分布**
- **各科目人數分布**

### 6.2 結果查看模式

**列表模式**（預設）：
- 表格顯示所有篩選學生
- 支援欄位排序：年級、姓名、學號、分數、狀態
- 支援關鍵字搜尋（姓名、學號）
- 可依年級、科目、狀態篩選

**分組模式**：
- 依「年級 + 班級」分組顯示
- 每組可折疊收起
- 適合按班級列印或核對

### 6.3 狀態標示

| 狀態 | 標示 | 說明 |
|------|------|------|
| 優先 | ⭐ 金色 | 現職學生名單中的學生 |
| 一般 | 綠色 | 符合篩選條件的一般學生 |
| 排除 | 灰色 | 特殊學生名單中（顯示但不計入）|

### 6.4 匯出功能

| 格式 | 說明 |
|------|------|
| **Excel (.xlsx)** | 含狀態標籤的完整表格，適合後續處理 |
| **CSV** | 通用格式，適合匯入其他系統 |

匯出欄位：年級、班級、座號、學號、姓名、身分證字號、科目分數、篩選科目、篩選分數、狀態

## 7. 資料格式規範

### 7.1 成績檔案格式

成績 Excel/CSV 必須包含以下欄位（欄位名稱可為中文或英文）：

**必要欄位**：
| 欄位 | 可接受的欄位名稱 |
|------|----------------|
| 學號/身分證 | 學號、身份證、身分證、ID、student_id |
| 姓名 | 姓名、name、學生姓名 |
| 班級代碼 | 班級、class、班別 |
| 分數 | 分數、score、成績、得分 |

**班級代碼格式**：`年級+班別`，例如：
- `203` = 二年級三班
- `301` = 三年級一班
- `101` = 一年級一班

**多工作表支援**：若 Excel 中有多個工作表，系統會尋找名稱含有科目關鍵字的工作表（國文/chinese、英文/english、數學/math/maths）。

### 7.2 分數範圍

系統預設合理分數範圍為 **0 ~ 150**，超出此範圍的分數會觸發異常警告。

### 7.3 輔助名單格式

現職學生 / 特殊學生名單只需包含：
- 身分證字號（或學號）欄位
- 姓名欄位（可選）

## 8. 智慧欄位偵測說明

系統採用**雙層偵測機制**：

### 第一層：標題行偵測

若檔案首行為標題行，系統對欄位名稱進行關鍵字比對：

```
「學號」「身份證」「身分證」「ID」→ 學號欄
「姓名」「name」「學生姓名」      → 姓名欄
「班級」「class」「班別」          → 班級欄
「分數」「score」「成績」「得分」  → 分數欄
```

### 第二層：內容特徵偵測

若無標題行，系統分析每欄的資料特徵：

| 特徵 | 判斷依據 |
|------|---------|
| 身分證字號 | 符合台灣身分證格式（1字母+9數字）|
| 班級代碼 | 3位數字，首位為年級（1-3），次位為班別（1-9）|
| 分數 | 數值介於 0–150 |
| 姓名 | 2–5個中文字（漢字）|

> 建議在 Excel 中保留標題行，可大幅提升偵測準確率。

## 9. 資料持久化（自動儲存）

### 自動儲存機制

系統使用瀏覽器的 **IndexedDB** 儲存所有工作資料，包括：

- 三科成績資料
- 現職/特殊學生名單
- 篩選條件設定
- 篩選結果
- 各檔案名稱

每次資料變更後，系統會自動在背景儲存，**不需手動點擊儲存**。

### 重新載入還原

重新整理頁面後，若偵測到已儲存的資料，頁面頂端會顯示**藍色提示橫幅**：

```
📂 已從上次工作階段還原資料（含 1234 位學生、設定 6 條篩選規則）
```

### 清除所有資料

點擊右上角的「**清除資料**」按鈕，可清除所有已儲存的資料並重新開始。

> **注意**：清除後無法復原，請確認後再操作。

## 10. 常見問題 FAQ

**Q：上傳 Excel 後沒有偵測到任何資料？**
> 請確認 Excel 中含有實際資料列（非空白工作表）。若使用多工作表，請確保含有科目關鍵字的工作表名稱（如「國文成績」、「English」）。

**Q：欄位偵測完全錯誤怎麼辦？**
> 上傳後在「預覽畫面」使用手動下拉選單重新對應每個欄位，再點擊「套用設定」。

**Q：班級代碼偵測錯誤（如 203 被解讀為分數）？**
> 請在手動調整中將對應欄位設定為「班級」。

**Q：篩選後發現部分學生跨科目 ID 不符合？**
> 進入步驟二，系統會在頁面頂端顯示跨科目 ID 不符合的警告清單。請回到步驟一確認原始資料是否有誤。

**Q：匯出的 Excel 亂碼？**
> 請使用 Microsoft Excel 或 LibreOffice 開啟 .xlsx 檔案。若使用 CSV 格式有亂碼，請在開啟時指定 UTF-8 編碼。

**Q：清除資料後，之前的篩選結果還能找回嗎？**
> 無法。IndexedDB 資料清除後不可復原。建議在清除前先匯出結果。

**Q：系統支援哪些瀏覽器？**
> 建議使用 Chrome 90+、Edge 90+、Firefox 88+ 或 Safari 15+。需支援 IndexedDB 與 ES2022 語法。
