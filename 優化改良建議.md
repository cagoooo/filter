### 2.2 大型 Excel 檔案造成 UI 凍結

**問題**：上傳超過 10,000 列的 Excel 時，主執行緒被 XLSX 解析佔用，UI 無法回應。
**影響**：使用者體驗極差，感覺系統當掉。

**修復方式**：將 Excel 解析移至 Web Worker

```typescript
// artifacts/grade-filter/src/lib/excel.worker.ts
import { parseExcelFile } from './excel'
self.onmessage = async (e: MessageEvent) => {
  const result = await parseExcelFile(e.data.buffer, e.data.subject)
  self.postMessage(result)
}
```

```typescript
// 在 FileUploadCard.tsx 中使用 Worker
const worker = new Worker(new URL('../lib/excel.worker.ts', import.meta.url), { type: 'module' })
```

### 2.3 篩選結果在班級代碼為空時顯示異常

**問題**：若學生資料中班級欄位為空，分組模式會產生「undefined 班」的分組。
**修復**：在 `ResultPage.tsx` 的分組邏輯加入 fallback 值。

```typescript
// 修改前
const groupKey = `${student.grade}年級${student.class}班`
// 修改後
const groupKey = `${student.grade ?? '未知'}年級${student.class ?? '未知'}班`
```

**UI 位置**：在篩選頁面頂端加入 `<TemplateSelector>` 元件。

## 4. P2 — 中期優化項目

### 4.1 更完善的錯誤處理與使用者提示

目前系統在部分異常情況下可能靜默失敗。建議加入全域錯誤邊界：

```typescript
// src/components/ErrorBoundary.tsx
class ErrorBoundary extends React.Component<Props, State> {
  componentDidCatch(error: Error, info: ErrorInfo) {
    console.error('Unhandled error:', error, info)
    // 可選：發送錯誤報告
  }
  render() {
    if (this.state.hasError) {
      return <ErrorFallback onReset={() => this.setState({ hasError: false })} />
    }
    return this.props.children
  }
}
```

### 4.2 欄位偵測置信度顯示

目前欄位自動偵測只顯示結果，不顯示置信度。建議在 UploadPreview 中顯示每個欄位的偵測置信度（高/中/低），讓使用者知道哪些欄位需要重點確認。

```typescript
interface ColumnDetectionResult {
  field: StudentField
  confidence: 'high' | 'medium' | 'low'
  reason: string  // 例如：「符合台灣身分證格式（100%）」
}
```

### 4.3 檔案格式預設設定儲存

若使用者的學校 Excel 格式固定，每次都需要手動調整欄位。建議增加「記住此次對應設定」功能：

```typescript
interface ExcelProfile {
  name: string  // 例如：「學校成績 Excel 格式」
  columnMapping: ColumnMapping
  hasHeader: boolean
}
// 儲存在 IndexedDB，下次上傳時自動套用
```

### 4.4 資料匯入 Undo/Redo

允許使用者在上傳後撤銷至上一個狀態（例如：不小心上傳錯誤的檔案後，可以一鍵回復）。

```typescript
// 在 AppContext 中加入 history stack
const [history, setHistory] = useState<AppState[]>([])
const undo = () => {
  const prev = history[history.length - 1]
  if (prev) setState(prev)
}
```

### 4.5 行動裝置 (RWD) 改善

目前使用 `use-mobile.tsx` hook 偵測行動裝置，但 UI 尚未針對小螢幕完全優化。

建議改進項目：
- FilterPage 的條件卡片改為垂直堆疊
- ResultPage 表格改為卡片列表（行動裝置）
- 上傳區域更大的點擊目標

### esbuild 平台問題（立即注意）

`pnpm-workspace.yaml` 中有以下設定：

```yaml
overrides:
  "esbuild": "npm:@esbuild/linux-x64@0.25.5"
```

這會強制在所有環境使用 Linux 版 esbuild。**在 Windows 開發環境可能造成問題**。

建議改為：

```yaml
# 移除 overrides 中的 esbuild 強制覆蓋
# 或改為條件式設定，只在 CI/Replit 環境使用 linux 版
```
