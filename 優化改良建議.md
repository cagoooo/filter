## 1. 專案現況摘要

### 現已上線版本
- **部署網址**：https://cagoooo.github.io/filter/
- **原始碼**：https://github.com/cagoooo/filter
- **部署方式**：GitHub Pages + GitHub Actions 自動化 CI
- **技術棧**：React 19.1 + Vite 7.3 + TypeScript 5.9 + Tailwind CSS v4 + wouter
- **資料儲存**：IndexedDB（完全離線可用，無需後端）

### 核心功能
1. **三步驟引導式流程**：匯入成績 → 設定篩選 → 查看結果
2. **智慧欄位偵測**：自動辨識學號、身分證、姓名、班級、座號、成績欄位
3. **多科成績匯入**：國文、英文、數學獨立 Excel 上傳
4. **優先名單 / 特殊生名單**：自動排除特殊生、標記優先錄取學生
5. **前/後百分比篩選**：支援資優（前 %）與學習扶助（後 %）雙模式
6. **結果匯出**：Excel 格式下載
7. **社群分享卡片**：LINE / FB / Twitter 預覽圖已配置完成

### 4.3 全域 Error Boundary

**場景**：目前若任何元件意外拋錯，整個應用會變成白屏，使用者不知如何恢復。

**新增檔案**：`src/components/ErrorBoundary.tsx`

```typescript
import { Component, ErrorInfo, ReactNode } from 'react'

interface Props { children: ReactNode }
interface State { hasError: boolean; error: Error | null }

export class ErrorBoundary extends Component<Props, State> {
  state: State = { hasError: false, error: null }

  static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error }
  }

  componentDidCatch(error: Error, info: ErrorInfo) {
    console.error('[ErrorBoundary]', error, info)
  }

  handleReset = () => {
    this.setState({ hasError: false, error: null })
  }

  render() {
    if (this.state.hasError) {
      return (
        <div className="min-h-screen flex flex-col items-center justify-center p-8">
          <h1 className="text-2xl font-bold mb-4">發生錯誤</h1>
          <p className="text-gray-600 mb-6">
            {this.state.error?.message ?? '未知錯誤'}
          </p>
          <button
            onClick={this.handleReset}
            className="px-6 py-2 bg-blue-500 text-white rounded"
          >
            重試
          </button>
          <button
            onClick={() => location.reload()}
            className="mt-2 text-sm text-gray-500 underline"
          >
            重新整理頁面
          </button>
        </div>
      )
    }
    return this.props.children
  }
}
```

在 `main.tsx` 包住 `<App />`，並考慮在每個頁面內層再包一層（讓一個頁面崩潰不影響其他頁面）。

**預估工作量**：2 小時

### 4.4 成績為非數字（缺考、補考）的處理

**場景**：學生缺考時 Excel 常填「缺」「補」「—」，目前被解析為 NaN 會導致篩選結果異常。

**修改位置**：`src/lib/excel.ts` 的分數解析邏輯

```typescript
function parseScore(raw: unknown): number | undefined {
  if (raw === null || raw === undefined || raw === '') return undefined
  const str = String(raw).trim()
  // 非數字視為缺考，不納入排序
  if (['缺', '補', '—', '-', 'N/A', 'NA'].includes(str)) return undefined
  const num = Number(str)
  if (isNaN(num)) return undefined
  if (num < 0 || num > 150) return undefined  // 異常分數保護
  return num
}
```

在結果頁面也要區分「0 分」和「未考」：
```tsx
{score === undefined ? '缺考' : score}
```

**預估工作量**：1.5 小時

export function useKeyboardShortcuts() {

      const mod = e.ctrlKey || e.metaKey

### 5.8 撤銷 / 還原（Undo/Redo）

**需求背景**：不小心上傳錯誤檔案或清除資料後，沒有復原機制。

**建議實作**：

```typescript
// AppContext 中加入歷史堆疊
const [history, setHistory] = useState<AppSnapshot[]>([])
const [future, setFuture] = useState<AppSnapshot[]>([])

const snapshot = (): AppSnapshot => ({
  chineseData, englishData, mathData,
  currentStudents, specialStudents,
  filterConfigs, filterResults,
  // ... 所有檔名
})

const pushHistory = () => {
  setHistory(h => [...h.slice(-19), snapshot()])  // 保留最近 20 步
  setFuture([])
}

const undo = () => {
  if (history.length === 0) return
  const prev = history[history.length - 1]
  setFuture(f => [snapshot(), ...f])
  setHistory(h => h.slice(0, -1))
  restoreSnapshot(prev)
}
```

**UI**：
- 右上角顯示 ↶ ↷ 按鈕
- 快捷鍵 `Ctrl+Z` / `Ctrl+Shift+Z`

**預估工作量**：6 小時

## 11. 風險提醒與注意事項

### 11.1 資料隱私與安全

⚠️ **重要**：本應用處理學生個資（姓名、身分證、成績），請注意：

- **不要將測試資料 commit 到 GitHub**（已建議 `.gitignore`）
- IndexedDB 資料僅存在瀏覽器，**清除瀏覽器資料會全部消失**
- 若需跨裝置同步，應使用**私人伺服器**而非公開雲端
- 建議定期提醒使用者匯出備份
- 若未來整合後端，需符合**個人資料保護法**要求

### 11.2 瀏覽器相容性

- IndexedDB：Chrome/Edge/Firefox/Safari 皆支援（IE 不支援）
- XLSX 解析：需現代瀏覽器
- **建議明確標註「僅支援 Chrome 100+ / Edge 100+ / Firefox 100+」**

### 11.3 資料量上限

- IndexedDB 單一網域通常限 50MB–2GB（依瀏覽器）
- 5,000 筆學生資料約佔 2–5MB，遠未達上限
- 若全縣市整合（數十萬筆），必須改用後端

### 11.4 依賴套件升級風險

| 套件 | 風險 | 建議 |
|------|------|------|
| React 19 | 新版生態仍在穩定中 | 保持關注，暫不大規模重構 |
| Tailwind v4 | 與 v3 API 不同 | 文件尚少，升級前詳閱 |
| Vite 7 | 相對新版 | 通常穩定，小心 plugin 相容性 |
| wouter | 輕量但社群小 | 若需路由守衛等進階功能，考慮 React Router |
| xlsx (SheetJS) | 社群版更新慢 | 考慮換用 ExcelJS |

### 11.5 Git Workflow 建議

目前為單人開發，建議未來加入：
- **分支策略**：`main`（穩定）/ `dev`（開發）/ `feature/*`（功能）
- **Commit 訊息規範**：使用 Conventional Commits（`feat:`、`fix:`、`docs:`、`refactor:`）
- **Pull Request Template**：檢查清單（測試、文件、螢幕截圖）
- **GitHub Actions**：自動執行測試、lint

### 11.6 部署相關

- GitHub Pages 免費版有**流量限制**（每月 100GB），學校規模綽綽有餘
- 若未來需要自訂網域（例如 `grade.school.edu.tw`），需在 DNS 加 CNAME
- HTTPS 自動啟用，無需額外設定
